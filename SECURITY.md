# Security Policy

## Supported versions

The default branch is the only actively maintained line unless a maintainer states otherwise.

## Reporting a vulnerability

- Do not open a public GitHub issue for security-sensitive reports.
- Send the report to the maintainers through the repository owner contact channel and include:
  - affected area
  - reproduction steps
  - impact assessment
  - suggested mitigation if available

## Scope notes

- This repository is a frontend application and depends on external backend APIs.
- Please clearly separate frontend-only issues from backend or infrastructure issues when reporting.

## Response expectations

Maintainers will aim to acknowledge valid reports within a reasonable time, assess severity, and coordinate a fix before public disclosure.
