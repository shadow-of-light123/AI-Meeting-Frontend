# AI 面试室构思板一期 PRD

## 1. 背景

当前 AI 面试室以“读题 -> 直接文本作答”为主。对于系统设计、项目排障、流程拆解、算法思路等题型，用户往往需要先列提纲、画调用链、梳理回答顺序。仅依赖文本输入会导致：

1. 用户认知负载高
2. 回答结构容易混乱
3. 面试节奏被频繁的删改文本打断

因此需要在面试室内提供一个“构思板”，让用户先构思，再回到回答区组织语言。

## 2. 一期目标

在不改动现有 React 架构的前提下，为面试室提供一个轻量、低门槛、可随时收起的构思板工具。

一期完成后，用户应能：

1. 在面试室中打开右侧构思板
2. 使用文本笔记快速列提纲
3. 使用简单画笔在画布上画流程/结构/调用链
4. 收起构思板后继续答题
5. 在同一 `sessionId` 下刷新页面后恢复之前的构思内容
6. 将笔记一键插入回答输入框，降低在两个区域之间来回复制的负担

## 3. 非目标

一期不做以下能力：

1. 不做多人协作
2. 不做图形库级别的复杂图元系统
3. 不做后端持久化
4. 不做 AI 自动解析草稿
5. 不做 PDF / 图片导出
6. 不做实时同步到报告页

## 4. 用户场景

### 场景 A：系统设计题

用户先在画板里画出模块边界、调用链和缓存层，再回到输入框用语言表达方案。

### 场景 B：项目复盘题

用户先在笔记区写出：

1. 背景
2. 问题
3. 方案
4. 数据结果

然后一键插入回答框，减少重复输入。

### 场景 C：排障题

用户先画问题定位路径，确认顺序，再口头或文本作答。

## 5. 前端方案评估

### 候选方案

1. `tldraw`
   - 优点：白板能力完整
   - 缺点：过重，一期明显超配

2. `@excalidraw/excalidraw`
   - 优点：体验成熟
   - 缺点：依赖重，接入成本高，和当前项目风格不完全一致

3. `react-konva`
   - 优点：对 Canvas 封装较好
   - 缺点：需要新增依赖，一期能力仍然偏重

4. 原生 `Canvas + React state + localStorage`
   - 优点：零新增依赖、体积小、可控性高、足够覆盖一期需求
   - 缺点：需要自己实现基础绘制逻辑

### 一期选择

选择：`原生 Canvas + 现有 Radix Sheet + localStorage`

原因：

1. 当前项目依赖里已有 `@radix-ui/react-dialog` 封装出的 `Sheet`
2. 一期只需要“笔记 + 轻量绘制”
3. 避免为了 MVP 引入高体积白板库
4. 更符合当前项目“轻改造、不大改架构”的原则

## 6. 功能范围

### 6.1 入口

在面试室顶部 Header 增加按钮：`构思板`

### 6.2 展现形态

右侧滑出式抽屉，宽度约 `420px`，不遮挡整个页面。

### 6.3 功能模块

#### 模块 A：提纲笔记

1. 多行文本输入
2. 自动保存到 localStorage
3. 一键清空
4. 一键插入回答框

#### 模块 B：轻量画板

1. 画笔
2. 橡皮
3. 线宽切换
4. 撤销
5. 清空
6. 自动保存到 localStorage

## 7. 数据持久化

存储方式：浏览器 `localStorage`

Key 设计：

`interview-sketchpad:v1:${sessionId}`

数据结构：

```json
{
  "notes": "问题拆解...",
  "strokes": [
    {
      "tool": "pen",
      "width": 3,
      "points": [
        { "x": 0.12, "y": 0.18 },
        { "x": 0.24, "y": 0.22 }
      ]
    }
  ],
  "updatedAt": 1710000000000
}
```

说明：

1. 坐标采用相对比例，便于画布尺寸变化后恢复
2. session 维度隔离，避免不同面试互相污染

## 8. 交互要求

1. 默认关闭
2. 点击 `构思板` 打开
3. 关闭时不丢失内容
4. 刷新后恢复当前 session 的内容
5. 插入回答框时：
   - 若输入框为空，则直接填入
   - 若输入框非空，则追加到末尾

## 9. UI 约束

1. 延续现有面试室的浅色、轻边框风格
2. 不引入“复杂设计工具”风格的密集按钮栏
3. 工具操作区压缩在顶部一行
4. 右侧抽屉中的“笔记区”优先于“画板区”，因为真实高频场景是列提纲，不是自由绘图

## 10. 技术拆分

### 10.1 新增组件

1. `InterviewSketchpadSheet`
2. `InterviewSketchpadCanvas`

### 10.2 新增状态能力

1. 当前构思板开关状态
2. 当前笔记内容
3. 当前画板 strokes
4. localStorage 读写与恢复

### 10.3 页面集成点

集成到：

1. `InterviewHeader`
2. `InterviewPage`

## 11. 验收标准

1. 面试室可打开构思板
2. 可输入笔记并自动保存
3. 可在画板中绘制、撤销、清空
4. 可将笔记插入回答框
5. 关闭/刷新后内容仍可恢复
6. 不影响现有问答、摄像头、神态评估、结束面试流程

## 12. 风险与后续

### 一期风险

1. 原生 Canvas 画布在高频绘制下需要控制重绘
2. localStorage 只适合单端暂存，不适合跨设备同步

### 二期方向

1. 画布导出图片
2. AI 根据草稿生成回答提纲
3. 后端持久化草稿
4. 更丰富的图元支持
